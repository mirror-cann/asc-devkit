# 精度转换<a name="ZH-CN_TOPIC_0000002603424969"></a>

## 二进制精度舍入规则

在了解不同类型之间的精度转换规则之前，需先了解二进制的精度舍入规则。

如下图所示，一个二进制数的尾部若干位可能因目标精度不足而被丢弃，这部分称为**待舍入部分**；剩余保留的部分称为**非舍入部分**。根据待舍入部分的值以及所选的**精度舍入模式**，决定非舍入部分的末位是否需要 **+1（进位）**。

**图1** 二进制精度舍入规则示意图<a name="fig1"></a>

![二进制精度舍入规则示意图](../../../figures/binary_rounding.png)

各精度舍入模式对待舍入部分的判断规则如下表所示：

**表1** 精度舍入模式<a id="tab1"></a>

| 模式 | 描述 |
| ---- | ---- |
| CAST_NONE | 当转换过程存在精度损失时，按CAST_RINT模式处理；当不存在精度损失时，不进行舍入。 |
| CAST_RINT | 向最近的偶数舍入。<br>&bull;若待舍入部分的首位为0，则不进位。<br>&bull;若待舍入部分的首位为1且后续位不全为0，则进位。<br>&bull;若待舍入部分的首位为1且后续位全为0：当非舍入部分的末位为0，则不进位；当非舍入部分的末位为1，则进位。 |
| CAST_FLOOR | 向负无穷大方向舍入。<br>&bull;若符号位（S）为0（正数），则不进位。<br>&bull;若符号位（S）为1（负数）：当待舍入部分全为0，则不进位；否则，进位。 |
| CAST_CEIL | 向正无穷大方向舍入。<br>&bull;若符号位（S）为1（负数），则不进位。<br>&bull;若符号位（S）为0（正数）：当待舍入部分全为0，则不进位；否则，进位。 |
| CAST_ROUND | 四舍五入。若待舍入部分的首位为0，则不进位；否则，进位。 |
| CAST_TRUNC | 截断模式。直接丢弃待舍入部分。 |
| CAST_ODD | 向最近的奇数舍入。<br>&bull;若待舍入部分全为0，则不进位。<br>&bull;若待舍入部分不全为0：非舍入部分的末位为1，则不进位；当非舍入部分的末位为0，则进位。 |
| CAST_HYBRID | 随机舍入，目前特指输出结果是hifloat8_t数据类型时，会用到的一种随机舍入。 |

## 精度转换规则

本节主要对不同数据类型之间精度转换时的舍入行为进行介绍。不同数据类型的表示方式请参考[内置数据类型](内置数据类型.md)。当源值超出目标类型的表示范围时溢出，溢出默认按照饱和模式处理。

将源值以int16_t格式存入目的操作数中，无精度损失。

**示例**：输入：1，输出：1。

### int16_t → half

将源值按照精度舍入模式取到half所能表示的数，以half格式存入目的操作数中。

**示例**：输入：$2^{12} + 2$，写成half的表示形式：$2^{12} \times (1 + 2^{-11})$，要求$E_v = 12 + 15 = 27$，$M_v = 2^{-11}$。

**图2** int16_t转half

![int16_t转half](../../../figures/int16_to_half.png)

由于half只有10bit尾数，因此灰色部分要进行舍入：

- **CAST_RINT模式**输入得尾数`0000000000`，$E_v = 27$，$M_v = 0$，最终结果为：$2^{12}$。
- **CAST_FLOOR模式**输入得尾数`0000000000`，$E_v = 27$，$M_v = 0$，最终结果为：$2^{12}$。
- **CAST_CEIL模式**舍入得尾数`0000000001`，$E_v = 27$，$M_v = 2^{-10}$，最终结果为：$2^{12} + 4$。
- **CAST_ROUND模式**舍入得尾数`0000000001`，$E_v = 27$，$M_v = 2^{-10}$，最终结果为：$2^{12} + 4$。
- **CAST_TRUNC模式**舍入得尾数`0000000000`，$E_v = 27$，$M_v = 0$，最终结果为：$2^{12}$。
