# HF32

## 特性说明

HF32的核心功能体现为：用户在输入矩阵A和B都为`fp32`数据类型的场景下，当配置开启HF32开关后，`fp32`数据在执行矩阵乘法运算前会被转换并舍入为HF32（half-fp32）格式。这种做法可使`Mmad`（`fp32`->`fp32`）接口的计算性能提升至原来的两倍，但会带来一定的精度损失。需要注意的是，尽管中间计算使用HF32格式，最终的运算结果仍以`fp32`格式输出，以保证后续处理的兼容性。

舍入模式说明：`fp32`至`HF32`转换过程中的舍入模式由`SetHF32TransMode`接口配置。当`HF32`模式开启且设置`TransMode`为`NEAREST_ZERO`时，则`FP32`将以向零靠近的方式四舍五入为`HF32`；若设置`TransMode`为`NEAREST_EVEN`时，则`FP32`将以最接近偶数的方式四舍五入为`HF32`。

## 使用示例

```cpp
using namespace AscendC::Te;

// 开启HF32模式
AscendC::SetHF32Mode(AscendC::HF32Mode::ENABLE);
// 设置转换模式为向零靠近的方式处理
AscendC::SetHF32TransMode(AscendC::HF32TransMode::NEAREST_ZERO);

// Mmad计算代码
MmadParams params(m, n, k, 0, true);
auto atom = MakeMmad(MmadOperation{}, MmadTraitDefault{}).with(params);
Mmad(atom, l0C, l0A, l0B);

// 关闭HF32模式
AscendC::SetHF32Mode(AscendC::HF32Mode::DISABLE);
AscendC::SetHF32TransMode(AscendC::HF32TransMode::NEAREST_EVEN);
```
