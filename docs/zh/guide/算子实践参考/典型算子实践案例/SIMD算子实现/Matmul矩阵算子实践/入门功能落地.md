# 入门功能落地<a name="ZH-CN_TOPIC_000000_ADD_ENTRY"></a>

本节使用基础API进行矩阵乘法的编程指导，介绍基于Ascend C编程模型和Cube计算单元实现矩阵算子的基本方法，算子中数据搬运和计算流水线通过事件同步进行协调，降低编程复杂度。

## 详细内容

详细内容请参考 [matmul样例](../../../../../../../examples/01_simd_cpp_api/00_introduction/02_matrix/matmul_basic_api)，该样例涵盖：

- 算子实现：Global Tensor与Local Tensor的使用、DataCopy数据搬运、Matmul计算
- 功能调试：printf格式化输出、DumpTensor张量数据打印
- 编译运行：bisheng编译与执行
- 性能调试：msOpProf工具上板与仿真性能采集

### 可优化方向分析

上述实现正确完成了矩阵乘功能，但在性能层面存在以下几类可优化的空间。

| 可优化方向 | 当前实现的问题 | 预期优化收益 |
|-----------|---------------|-------------|
| **L1/L0单缓冲区** | A、B矩阵在L1和L0中各只有一份缓冲区，下一轮数据必须等上一轮消费完才能搬入 | 改为Ping-Pong双缓冲，让数据搬运和计算重叠执行 |
| **MTE2搬运粒度小** | 每次DataCopyIn搬入一个baseM×baseK小块，MTE2指令发射密集 | 增大搬运粒度，减少MTE2指令次数，提高带宽利用率 |
| **流水线串行执行** | 当前实现中使用PipeBarrier和SetFlag/WaitFlag进行流水线同步，但未采用双缓冲机制，MTE2、MTE1、M、FIX各硬件单元无法充分流水并行 | 通过双缓冲和硬件事件同步，使四个单元流水并行，显著提升吞吐 |
| **MTE1指令队列压力** | dav-2201下LoadData2D需for循环多次发射，MTE1指令队列深度有限 | 使用更高效的Load方式，降低MTE1队列压力 |
| **多核切分固定** | 硬编码`M / singleCoreM`计算核数分配，仅适用于固定矩阵规模和核数场景 | 通用切分策略，支撑任意核数和矩阵规模的自动分配 |
| **Tiling参数运行时计算** | 核内使用Scalar逐级计算各维度参数，引入额外指令开销 | 编译期确定参数，减少运行时开销 |
| **Fixpipe与Cube串行** | L0C 写回必须等Mmad全部完成后才开始，写回期间Cube空闲 | 细粒度Fixpipe发射，令写回与计算流水重叠 |
| **B矩阵搬运顺序** | A、B数据依次等量搬运，较大的矩阵场景下B数据到位晚可能成为流水线瓶颈 | 优先搬运B矩阵，缓解数据紧张 |
| **L2Cache未充分利用** | 当前实现中B矩阵每次均从GM读取，多核反复访问GM导致MTE2带宽压力大；L2Cache纯读带宽约3~4倍于GM，但当前实现未能利用这一特性 | 沿M方向切分A矩阵，使B矩阵跨轮驻留L2Cache，减少GM访问次数，有效降低MTE2耗时 |

> 以上各优化方向的详细实现方案，请参见[中级性能优化](./中级性能优化.md)和[高级极限打磨](./高级极限打磨.md)。
