# 同步机制<a name="ZH-CN_TOPIC_0000002554691683"></a>

SIMT是一种单指令多线程的编程模型，该编程模型旨在通过多线程并发执行达到隐藏访存延迟，提升计算吞吐量的目的。SIMT编程模型中，一个线程块(Thread Block)内的多个线程(Thread)并行执行，不同线程的执行进度和内存访问到达顺序可能不同。当线程之间存在数据依赖时，需要使用同步接口或内存栅栏接口约束执行顺序和内存可见性，避免读到未完成写入的数据。同步接口主要分为两类：

- **同步屏障(barrier)**：要求同一作用范围内的线程都到达指定位置后，程序才能继续执行。
- **内存栅栏(memory fence)**：约束调用线程在栅栏前后的内存访问顺序，使栅栏前的内存操作按指定范围对其他线程可见。内存栅栏不会等待其他线程到达同一位置。

## 接口概览

| 类型 | 代表接口 | 典型用途 |
| --- | --- | --- |
| 同步屏障 | `asc_syncthreads` | 线程块内阶段同步，例如先写Unified Buffer，再统一读取。 |
| 内存栅栏 | `asc_threadfence_block`、`asc_threadfence` | 发布数据或标志前，约束当前线程的内存访问顺序和可见性。 |

同步接口和内存栅栏接口的详细语义、作用范围、使用示例及注意事项，请参见[同步与内存栅栏](https://gitcode.com/cann/asc-devkit/blob/9.1.0/docs/api/SIMT-API/同步与内存栅栏/同步与内存栅栏.md)。

## 使用建议

同步屏障适用于需要线程块内所有线程完成某一阶段后再继续执行的场景，内存栅栏适用于只需约束当前线程内存写入顺序和可见性的场景，两者均不能替代原子操作保证读改写过程的原子性。关于各接口的作用范围、代码示例和详细使用建议，请参见API参考[同步与内存栅栏简介](../../../api/SIMT-API/同步与内存栅栏/同步与内存栅栏简介.md)；多个线程并发写同一地址需要避免写冲突时，请参见[原子操作](原子操作.md)。
